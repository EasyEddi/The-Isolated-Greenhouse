# Eure Aufgabe: Das Capstone-Projekt 🚀 (Woche 2 & 3)

In Woche 1 hat jeder von euch **alleine** eine App gebaut und geshippt — stark! 🎉
Jetzt kommt das große Finale: Ihr baut **zusammen EIN größeres Produkt.** Euer Capstone.

Das Neue daran ist nicht schwererer Code — sondern **als Team an einem Projekt arbeiten.** Genau so, wie ein echtes Startup-Team es macht. Am Ende zeigt ihr euer Produkt allen in eurem Bekanntenkreis. 😎

---

## Wie ihr als Team arbeitet (das Wichtigste!)

- **Ein gemeinsames Repo** — ihr beide arbeitet im selben Projekt.
- **Jeder auf seinem eigenen Branch** — so kommt ihr euch nicht in die Quere.
- **Pull Requests + Review:** Bevor etwas ins Hauptprojekt kommt, schaut der andere einmal drüber und merged es. So landet nie kaputter Code im Projekt. Codex kann euch dabei helfen "Erstelle einen PR mit meinen Änderungen" oder "Erkläre und reviewe die Änderungen in PR #xx". Immer sicher gehen, dass der main branch funktioniert.
- **Aufteilen mit Issues:** Jede Aufgabe ist ein GitHub Issue mit einem Namen dran — du weißt immer, was *du* gerade machst.
- **Standup:** Jeden Morgen 5 Minuten mit Daniel — *Was habe ich gestern gemacht? Was mache ich heute? Wo hänge ich?*

> 💡 Goldene Regel: **Klein halten.** Lieber wenige Dinge, die richtig gut funktionieren, als viele, die halb fertig sind.

## Was ihr in diesen zwei Wochen lernt
- Im **Team** an einem Projekt arbeiten (Branches, Pull Requests, Code-Review)
- Ein größeres Projekt **planen und aufteilen**
- Eine echte **Technik-Entscheidung** treffen (Frontend vs. Backend+Datenbank)
- Den Profi-Loop leben: **bauen → testen → Feedback → verbessern → shippen**
- Euer Produkt vor Publikum **präsentieren**
